  2. Javascript: Novice to Ninja - Chapter 12: Object Oriented Programming in JS
    1. (E5) Constructor Function: 
      const Dice = function(sides=6){
    this.sides = sides;
    this.roll = function() {
        return Math.floor(this.sides * Math.random() + 1)
    }
}
    2. const redDice = new Dice; vs. const redDice = new Dice(); :: No parenthesis if constructor functions exists.
    3. (E6) Class declaration instead of "(E5) Constructor Function": 
      class Dice {
    constructor(sides=6) {
        this.sides = sides;
    }
    roll() {
        return Math.floor(this.sides * Math.random() + 1)
    }
}
    4. Not sure why I would want this but you can use an object with the .constructor(var) syntax to create another instand of an object...: Example: const greenDice = new redDice.constructor(10);
    5. The book is a little confusing but prototype is sort of like static... all objects created from the class will have the same prototype and value. Changing it for one changes it for all (see below). Then there are the usual class properties/methods that are extended to the instances/objects but those are unique to each obj/instance.
    6. The whole prototype system is a little finiky. You can't really change them easily, constructors mess everything up. And it seems that when they can be changed it doesn't change them for ALL but rather for subsequent.
    7. Property Attributes: value, writable, enumerable, configurable
    8. Enumerable: this boolean value shows whether a property will show up when the object is displayed in a for in loop, and is false by default
    9. "Nesting" this: that = this: 
      superman.findFriends = function(){
    const that = this;
    this.friends.forEach(function(friend) {
        console.log(`${friend.name} is friends with ${that.name}`);
    }
    );
}
    10. Alt "Nesting" this: bind(this) 
      superman.findFriends = function() {
    this.friends.forEach(function(friend) {
        console.log(`${friend.name} is friends with ${this.name}`);
    }.bind(this);)
}
