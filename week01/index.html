Web Frontend Development II

Week 01

Notes:

  1. Doing Stuff with Web Things:
    1. Think. Decide what products/technology you need.
    2. Design. Plan, plan, plan
    3. Test. This isn't exactly running tests. This is planning the tests you will need to run to make sure it is functioning properly. Sort of like a science experiment. Designing the procedure/test so you know you can get the data and see the results.
    4. Create. Start the actual work. Run into problems? Return to step 1 and repeat.
    5. Web Colors
    6. localStorage documentation
  2. Master Mobile UX:
    1. Mobile first; of course.
    2. Shifting access buttons to the bottom where fingers can reach.
    3. Hamburger button is not as well known yet, meanwhile the word menu within something that makes it look like a button still receives the most clicks/attention from users.
    4. Floating labels within the content of an input. I.e. remove labels, and then when the user clicks in a field the hint text also displays as a label at the top (but inside) the input.
    5. Make larger check boxes...
    6. Don't mask the password field. Instead add the button to hide/show it as needed. Again default is visible.
    7. Anchor tag phone numbers... href="tel:12223334444". This hint is likely mostly for mobile devices.
    8. Idea: use paralax design to place an add behind the regular content. Grants a small window for the add to be peeped at...

Questions:

  1. Doing Stuff with Web Things:
    1. What section am I supposed to be reading? The instructions don't appear to match the reading.
    2. Where does it store the localstorage key value pairs?
    3. How do you retrieve them? localStorage.getItem(value)
  2. Master Mobile UX:
    1. Moving menus to the bottom brings them closer to the fingers/thumb in particular, but doesn't that also make it easier to accidentally click them lowering the user experience?
    2. Follow-up: Are there statistics to show what the trade-off is? Or some other guidelines to adjust this format so that it works better?
