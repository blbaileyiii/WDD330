Week 03

Notes:

  1. Object Method: this
    1. Arrow Function has no "this". When "this" is ref. it ref. to next higher level.
  2. Javascript: Novice to Ninja Ch5: Objects
    1. If the name of the variable and the name of the property are the same, then you can use shortform obj creation:
      const name = 'Iron Man';
      const realName = 'Tony Stark';
      // long way
      const ironMan = { name: name, realName: realName };
      // short ES6 way
      const ironMan = { name, realName };
    2. You can use dot notation or [""]; [] is "superior" in that it can be used to pass variables, concat strings, etc, to form the prop.
    3. Loop over properties: for(const key in object)
    4. Object.keys(obj) returns array of keys and Object.values(obj) returns array of values.
    5. Namespacing: const namespaceVarName = {}; toss the functions into that and separate by comma. Call the same way you do an obj. Helps to scope variables and avoid collisions.
  3. Javascript: Novice to Ninja Ch6: The Document Object Model
    1. DOM isn't javascript: "document" is. The term "document" is used to access the DOM which doesn't care what language you are using.
    2. CSS property background-color becomes backgroundColor in the DOM. Dashes are not legal characters in property names.
  4. Javascript: Novice to Ninja Ch7: Events
    1. document.body.addEventListener("click", doSomething);
    2. Using inline is not longer recommented: you want to keep javascript in javascript, css in css, and html in html. Other reasons exist. This is the one I like the best, though single vs. multiple event handlers seems to be another.
