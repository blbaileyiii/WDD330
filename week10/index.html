  1. MDN: Validating forms
    1. Warning: Never trust data passed to your server from the client. Even if your form is validating correctly and preventing malformed input on the client-side, a malicious user can still alter the network request.
    2. There are two basic form validation styles: A) Built in Form Validation, and B) Javascript
    3. Form Validation that is built in (this is when you set up the Form elements like an input being required or of type email.)
    4. Javascript (again, two types, intercepting the form submission and/or setting up various input events (ondirty, onchange, etc.))
    5. CSS elements :valid/:invalid will work with native form validation. Javascript can also be used to trigger the valid and invalid state of an input triggering the css as well. It just takes extra steps.
    6. The developer should take care to inform the user what the constraints are. That provides a better user experience. Example: Input A) has a max length of 8. Good: Tell the user that the entry can't be more than 8 characters. Best: Tell the user how many characters they have left...
    7. NOTE: There is a built in API supported on most browsers for Javascript validation. Using this will produce the same general results as using the built in form validation. However if you want to have your own notifications and styling etc, then you can just build custom.
    8. NOTE/WARNING: setCustomeValidity is not JUST setting the message IF that happens. It isn't even like an alert that only happens IF it happens. Instead AS long as the message is set the field is treated as invalid. You must setCustomValidity back to "" in an else/default/etc path for cases when the input is valid.
  2. MDN: Using Fetch
    1. Retreive Data and put it in a json format 
      fetch('http://example.com/movies.json')
.then(response => response.json())
.then(data => console.log(data));
    2. Fetch requests are controlled by the connect-src directive of Content Security Policy rather than the directive of the resources it's retrieving.
    3. Upload JSON data... 
      const data = { username: 'example' };

fetch('https://example.com/profile', {
    method: 'POST', // or 'PUT'
    headers: {
    'Content-Type': 'application/json',
    },
    body: JSON.stringify(data),
})
.then(response => response.json())
.then(data => {
    console.log('Success:', data);
})
.catch((error) => {
    console.error('Error:', error);
});
