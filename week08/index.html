Notes:

  1. HTML5 & CSS3 for the Real World - Chapter 8: CSS3 Transforms and Transitions
    1. The order of transform functions makes a difference. Ex: if you rotate before translating, your translate direction will be on the rotated axis.
    2. Transistion: Any property changing from one value to another for which you can find a valid midpoint can be transitioned.
    3. Visibility can also "transition" though it is more like an on/off switch. Probably better to set it to visible and adjust using opacity.
    4. Negative delays are weird...just keep that in mind. You can end up jumping % through a transition
  2. HTML5 & CSS3 for the Real World - Chapter 12: Canvas, SVG, and Drag and Drop
    1. Canvas: WARNING: You've got to specify width and height using HTML first. That determines the canvas size. If not then it defaults to 300x150 because CSS is applied AFTER loading in the element.
    2. Canvas is very similar to SVG, except it is created using JS. You declare the HTML element, then grab it from the DOM and then build it with JS.
    3. There is a lot to cover on Canvas. Revisit this chapter.
    4. Raphaël helps with SVG creation.
    5. Set the draggable attribute on any HTML elements you’d like to be draggable.
    6. Add an event listener for the dragstart event on any draggable HTML elements.
    7. Add an event listener for the dragover and drop events on any elements that you want to have accept dropped items.
    8. Note that draggable isnota Boolean attribute, so you have to explicitly set it to true.
